# הוראות הגדרת Supabase - Royal Cash App

## 1. כיבוי אימות אימייל (מומלץ לבדיקה)

אם אתה רוצה שהמשתמשים יוכלו להיכנס מיד אחרי הרשמה ללא אימות אימייל:

1. לך ל-Supabase Dashboard
2. בחר את הפרויקט שלך
3. לך ל-Authentication → Settings
4. מצא את "Enable email confirmations"
5. **כבה** את זה (Toggle OFF)
6. שמור

## 2. הגדרת URL לאימות אימייל (אם אתה רוצה להשאיר אימות)

אם אתה רוצה להשאיר אימות אימייל, צריך להגדיר את ה-URL הנכון:

1. לך ל-Supabase Dashboard
2. בחר את הפרויקט שלך
3. לך ל-Authentication → URL Configuration
4. ב-"Site URL" הכנס:
   - `http://localhost:5500` (אם אתה משתמש ב-Live Server)
   - `http://localhost:3000` (אם אתה משתמש ב-Port 3000)
   - או ה-URL של האתר שלך אם הוא כבר באוויר
5. ב-"Redirect URLs" הוסף:
   ```
   http://localhost:5500/**
   http://localhost:3000/**
   ```
   (הוסף את כל ה-URLs שאתה משתמש בהם)
6. שמור

## 3. בדיקה שהכל עובד

1. רענן את האפליקציה
2. נסה להירשם
3. אם כיבית אימות אימייל - אתה אמור להיכנס מיד
4. אם השארת אימות אימייל - תקבל מייל, לחץ על הקישור, ואז תוכל להתחבר

## 4. פתרון בעיות

### אם קישור האימות מוביל ל-localhost שלא עובד:
- הגדר את ה-URLs ב-Authentication → URL Configuration (כמו בסעיף 2)
- או כבה אימות אימייל (כמו בסעיף 1)

### אם אתה לא יכול להיכנס אחרי הרשמה:
- נסה להתחבר ידנית עם האימייל והסיסמה
- בדוק ב-Console (F12) אם יש שגיאות

### אם יש שגיאת RLS:
- הרץ את `quick_fix.sql` ב-SQL Editor
- ודא שה-policies מוגדרים נכון
